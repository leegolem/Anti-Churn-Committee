웰니스 앱 `헤이빗`의 30일 리텐션이 왜 떨어지는지 데이터로 원인을 찾고, 다음 분기에 실행할 개선 실험까지 제안하는 분석 과제


## 헤이빗(Haybit)은?
가상 기업 헤이빗(Haybit) 은 수면, 운동, 마음챙김, 식단 루틴을 관리하는 웰니스 앱이다.   
최근 신규 유입과 MAU는 늘고 있는데, 30일 리텐션은 계속 하락하고 있는 상황.

현 상황을 정리하면
> 최근 신규 유입과 MAU는 늘고 있는데, 30일 리텐션은 계속 하락하고 있지??

PM은 어느 기능이나 구간에서 이탈이 생기는지 알고 싶어 하고, 
대표는 투자사 보고에 쓸 데이터 기반 개선 방향이 필요한 상황이다.

# 주최측에서 제안한 핵심 가설
"루틴 챌린지(그룹) 기능을 첫 7일 내 경험한 사용자는 그렇지 않은 사용자보다 30일 리텐션이 유의하게 높으며, 이 차이는 알림 발송 빈도보다 온보딩 단계의 기능 경험에 더 크게 기인한다."

- 첫 7일 내 챌린지 경험자 vs 미경험자 : D30 리텐션 차이
- 알림 많이 받은 사람 vs 적게 받은 사람 : 알림 빈도가 리텐션에 영향을 주는지
- 온보딩 완료/초기 기능 경험 여부 : 초반 경험이 잔존에 더 중요한지

**알림을 많이 보내서 붙잡는 게 중요한가, 아니면 초반에 핵심 기능을 경험시키는 게 중요한가?**

# 해야하는 분석 방향
기준 지정
1. Day 0: 가입일자 기준으로 할지, 온보딩 완료일 기준으로 할지
2. D30 리텐션: 가입 후 30일째 앱실행이 있으면 잔존으로 볼지
3. 활성/비활성 유저: `앱실행` 이벤트 기준으로 볼지
4. 챌린지 경험: `챌린지_탐색` 또는 `챌린지참여` 중 무엇을 기준으로 볼지

지금 당장 생각나는 eda 방향성
1. 월별 가입자 / MAU / D30 리텐션 : 실제 하락 추세 확인
2. 가입월 코호트별 D30 리텐션 : 어느 가입월부터 리텐션이 나빠졌는지
3. 기간동안의 행동 분석: 남는 유저와 떠나는 유저의 초기 행동 차이 확인
4. 챌린지 경험 여부별 리텐션 : 주최측에서 제안한 핵심 가설 검증
5. 알림 수신/오픈 여부별 리텐션 : 알림이 효과적인지 확인
6. 가입경로/기기별 리텐션 : 특정 유입 채널이나 기기 문제가 있는지 확인
7. 온보딩 완료 여부별 리텐션 : 초반 정착 과정의 영향 확인

가설 검증 및 전략 도출
1. 몰?루
2. 몰?루
3. 핵심가설 검증에 따른 전략 도출
3. 온보딩 완료자의 리텐션이 높음 : 온보딩 플로우 개선
4. 챌린지 경험자의 리텐션이 낮음 : 챌린지 구성 개선



# 최종정리
1. 문제정의
    - MAU는 증가하지만 D30 리텐션이 하락한다.
2. 데이터 기준 설정
    - D30, 활성 유저, 챌린지 경험, 온보딩 완료 기준을 정의한다.
3. 리텐션 하락 구간 확인
    - 월별/코호트별로 언제부터 떨어졌는지 본다.
4. 초기 행동 차이 분석
    - 잔존 유저와 이탈 유저의 첫 7일 행동을 비교한다.
5. 가설 검증
    - 챌린지 경험, 온보딩 완료, 알림 빈도/오픈 여부 중 무엇이 더 설명력이 큰지 본다.
6. 전략 제안 및 실행안
7. 실행안


> 리텐션이란? : 가입한 유저가 시간이 지나도 다시 앱을 쓰고 있는 비율
> MAU란? : Monthly Active Users. 즉, 한 달 동안 앱을 한 번이라도 사용한 유저 수
> 코호트란?: 같은 기준으로 묶은 사용자 그룹
```
예시
가입월 기준 - 1월 가입자, 2월 가입자, 3월 가입자
가입경로 기준 - 오가닉 유저, 광고 유입 유저
행동 기준 - 첫 7일 내 챌린지 참여자, 미참여자
기기 기준 - iOS 유저, Android 유저
으로 분류가 가능.
즉, 같은 기준으로 묶인 유저들이 시간이 지나면서 얼마나 남는지 보는 분석
```
