# 웰니스 앱 리텐션 분석 - EDA 정리 (지륜)

## 분석 목표
헤이빗 앱의 30일 리텐션 하락 원인 파악을 위한 탐색적 데이터 분석 (EDA)

---

## 데이터 개요

| 파일 | 행 수 | 주요 컬럼 |
|------|-------|-----------|
| `01_User_Profile.csv` | 12,500명 | User_ID, 가입일자, 가입경로, 기기, 알림수신동의여부 |
| `02_Event_Log.csv` | 1,757,262건 | User_ID, Event_Time, Event_Type, Session_ID, 알림_유형 |

- **분석 기간**: 2025-01-01 ~ 2025-06-30
- **통합 데이터**: event left join user → 10개 컬럼

---

## 분석 흐름 (노트북 셀 순서)

### 1. 환경 설정
- pandas, numpy, matplotlib 임포트
- Windows/Mac/Linux 환경별 한글 폰트 자동 설정

### 2. 데이터 로드 & 타입 변환
- CSV 로드 (UTF-8-BOM 인코딩)
- `가입일자`, `Event_Time` → `datetime64` 변환

### 3. 기본 데이터 탐색
- shape, dtypes, 결측치 확인
- user: `알림수신동의_변경일자` 84% 결측 (변경 이력 없는 유저)
- event: `알림_유형` 87.5% 결측 (알림 관련 이벤트에만 존재)

### 4. 데이터 병합
- event 기준 left join → df (1,757,262행 × 10컬럼)
- 중복 행 없음 확인

### 5. 전처리
결측치를 "왜 없는지" 원인을 먼저 파악한 후 처리 방식을 결정함.

| 처리 항목 | 내용 |
|---------|------|
| `Event_Type` 결측 (26,456건) | 행 제거 → 1,730,806행 잔여 |
| `Session_ID`, `알림_유형` 결측 | `'없음'` 채움 (알림 이벤트 구조적 결측) |
| `가입경로`, `기기` 결측 | `'미상'` 채움 |
| `알림수신동의여부` 결측 | `False` 채움 (보수적 처리) |
| `알림수신동의_변경일자` | `str` → `datetime64` 변환, `NaT` 유지 |
| 로그 장애 구간 마킹 | `is_log_anomaly` 컬럼 추가 (3월 10~14일 플래그) |
| 완전 미활동 유저 확인 | 47명 (이벤트 기반 분석에서 자동 제외) |

### 6. 기초 통계
- 총 유저: 12,453명 / 총 이벤트: 1,730,806건
- 유저당 이벤트 중앙값 51건 (평균 139건, 최대 687건)
- 이벤트 종류: 앱실행 > 수면기록 > 알림수신 > 운동기록 순

### 7. 일별 이벤트 추이
- 전체 기간 일별 이벤트 수 꺾은선 그래프
- **3월 10~14일 로그 수집 장애 → 이벤트 약 70% 누락** (빨간 음영 표시, 공모전 측에서 확인)
- 5월 이후 서서히 하락 추세 감지

### 8. 월별 가입자 & 가입경로 분석
- 월별 신규 가입자 막대그래프
- **2월 가입자 4,384명으로 전월 대비 약 2배 폭증**
- 광고/오가닉 비율 변화 없음 → 외부 바이럴·언론 보도 등 자연 인지도 상승 가능성

### 9. 이벤트 종류별 분포
- 가로 막대그래프 + 비율(%) 텍스트
- **온보딩 완료율 45.7%**: 신규 유저 절반 이탈
- **알림 오픈율 10.9%**: 알림 콘텐츠 효과 저조

### 10. 유저 세그먼트 분포
- 파이 차트 3개 (가입경로 / 기기 / 알림 동의)
- 퍼포먼스광고 55.4% vs 오가닉 44.6%
- iOS 58.0% vs Android 42.0%
- 알림 동의 64.5% vs 미동의 35.5%
